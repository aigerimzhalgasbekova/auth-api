# OAuth 2.0 Authorization Code Flow with PKCE

This module implements the OAuth 2.0 authorization code flow with PKCE (Proof Key for Code Exchange) as described in [RFC 7636](https://tools.ietf.org/html/rfc7636).

## Features

- **Authorization Code Flow with PKCE**: Implements the secure authorization code flow with PKCE support
- **Self-Encoded Authorization Codes**: Authorization codes are implemented as JWT tokens signed with AWS KMS
- **Parameter Validation**: Comprehensive validation of all required OAuth 2.0 parameters
- **Error Handling**: Proper OAuth 2.0 error responses as per the specification
- **Security**: Uses AWS KMS for cryptographic operations

## Authorization Endpoint

The `/authorize` endpoint accepts the following parameters:

### Required Parameters

- `response_type`: Must be `"code"`
- `client_id`: The client identifier
- `redirect_uri`: The redirect URI where the authorization code will be sent
- `code_challenge`: The PKCE code challenge
- `code_challenge_method`: The PKCE code challenge method (`"S256"` or `"plain"`)

### Optional Parameters

- `state`: A random string to prevent CSRF attacks
- `scope`: The requested scopes (space-separated)

### Example Request

```
GET /authorize?response_type=code&client_id=my-client&redirect_uri=https://example.com/callback&code_challenge=E9Melhoa2OwvFrEMTJguCHaoeK1t8URWbuGJSstw-cM&code_challenge_method=S256&state=xyz123&scope=read+write
```

### Response

On success, the server returns a 302 redirect to the `redirect_uri` with the authorization code:

```
HTTP/1.1 302 Found
Location: https://example.com/callback?code=eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCIsImtpZCI6ImFsaWFzL3NpZ25pbmcta2V5In0...&state=xyz123
```

The response body also contains the authorization code and state:

```json
{
  "code": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCIsImtpZCI6ImFsaWFzL3NpZ25pbmcta2V5In0...",
  "state": "xyz123"
}
```

## Authorization Code Format

Authorization codes are implemented as self-encoded JWT tokens containing:

- `client_id`: The client that requested the code
- `redirect_uri`: The redirect URI used in the request
- `scope`: The granted scopes
- `code_challenge`: The PKCE code challenge
- `code_challenge_method`: The PKCE code challenge method
- `exp`: Expiration time (10 minutes from issuance)
- `iat`: Issued at time
- `jti`: Unique identifier for the code

## Error Responses

The server returns proper OAuth 2.0 error responses:

### 400 Bad Request

```json
{
  "error": "invalid_request",
  "error_description": "Missing required parameter: response_type"
}
```

### 401 Unauthorized

```json
{
  "error": "access_denied",
  "error_description": "User authentication required"
}
```

### 500 Internal Server Error

```json
{
  "error": "server_error",
  "error_description": "Internal server error"
}
```

## Environment Variables

- `KMS_KEY_ALIAS_NAME`: The AWS KMS key alias used for signing authorization codes

## Security Considerations

1. **PKCE Support**: Implements PKCE to prevent authorization code interception attacks
2. **Short-lived Codes**: Authorization codes expire after 10 minutes
3. **Unique Identifiers**: Each code has a unique identifier to prevent replay attacks
4. **KMS Signing**: Uses AWS KMS for cryptographic operations
5. **Parameter Validation**: Validates all input parameters to prevent injection attacks

## Integration

This authorization code flow module is designed to work with the existing token issuer and authorizer modules in the auth-api project. The authorization codes generated by this endpoint can be exchanged for access tokens using the token endpoint.
